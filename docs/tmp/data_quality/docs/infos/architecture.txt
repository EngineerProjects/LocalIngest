                              1. OBJECTIF DU PROJET
════════════════════════════════════════════════════════════════════════════════

Le projet vise à détecter les anomalies et incohérences dans les données de 
localisation des sites (points de gestion) issus de Proxima, fusionnés avec 
un datamart.

OBJECTIFS PRINCIPAUX :
──────────────────────
• Vérifier que chaque site peut être LOCALISÉ (via GPS ou adresse)
• Valider le FORMAT des données présentes (coordonnées GPS, code postal)
• Détecter les INCOHÉRENCES entre les données (GPS vs Pays/Département, CP vs Ville)
• Identifier les CAUSES PROBABLES des erreurs GPS (inversion X/Y, signe erroné)
• Générer un RAPPORT EXCEL détaillé pour faciliter la correction des données

PRINCIPE FONDAMENTAL :
──────────────────────
Un site est considéré "localisable" s'il possède AU MOINS UN moyen de 
localisation valide :
    1. GPS valide (prioritaire) : coordonnées complètes, format OK, dans le bon pays
    2. Adresse valide (fallback) : CP + Ville présents et cohérents

Si le GPS est valide → le site est localisé, pas besoin de vérifier l'adresse.
Si le GPS est invalide/absent → on vérifie l'adresse comme solution de repli.

│   │   └── regions.geojson            # Obligatoire pour validation GPS/Région

NIVEAUX DE SUPPORT PAR PAYS :
─────────────────────────────

┌─────────────────┬─────────────────────────────────────────────────────────────┐
│ Niveau          │ Description                                                 │
├─────────────────┼─────────────────────────────────────────────────────────────┤
│ COMPLET         │ Dossier references/{PAYS}/ existe avec :                    │
│                 │   • codes_postaux.csv                                       │
│                 │   • regions.geojson (ou departements/provinces/etc.)        │
│                 │ → Toutes les validations possibles                          │
├─────────────────┼─────────────────────────────────────────────────────────────┤
│ PARTIEL         │ Pas de dossier pays, mais pays présent dans                 │
│                 │ bbox_pays_global.json                                       │
│                 │ → Validation GPS vs pays uniquement                         │
│                 │ → Pas de validation CP/Ville ni GPS/Région                  │
├─────────────────┼─────────────────────────────────────────────────────────────┤
│ NON SUPPORTÉ    │ Pays absent de bbox_pays_global.json                        │
│                 │ → Aucune validation géographique possible                   │
│                 │ → Anomalie R-01 (INFO)                                      │
└─────────────────┴─────────────────────────────────────────────────────────────┘


LOGIQUE DE CHARGEMENT AU DÉMARRAGE :
────────────────────────────────────

1. Charger bbox_pays_global.json (fallback)
2. Lister les pays uniques dans les données source
3. Pour chaque pays unique :
   a. Vérifier si references/{PAYS}/ existe
   b. Si OUI → charger codes_postaux.csv et regions.geojson
   c. Si NON → vérifier présence dans bbox_pays_global.json
   d. Stocker le niveau de support (COMPLET / PARTIEL / NON SUPPORTÉ)

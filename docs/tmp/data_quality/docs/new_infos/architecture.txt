================================================================================
               ARCHITECTURE SIMPLIFIÉE — CONTRÔLE QUALITÉ LOCALISATION
================================================================================

OBJECTIF
--------
Remplacer la logique actuelle trop proche d'un géocodage complet par un contrôle
simple, lisible et exploitable en correction métier.

Le système ne cherche plus à "localiser parfaitement" le site.
Il répond seulement à une suite courte de questions métier.


PRINCIPE GÉNÉRAL
----------------

1. Filtrer les sites actifs.
2. Contrôler le GPS si les coordonnées sont présentes.
3. Si le GPS ne permet pas de valider le site, basculer sur l'adresse.
4. Produire une seule synthèse par site.


RÈGLES MÉTIER RETENUES
----------------------

Ordre de contrôle :

1. Vérifier si longitude + latitude sont renseignées.
2. Si oui, vérifier si le point est dans la bbox du pays.
3. Si oui, vérifier si le point est cohérent avec le code postal
   quand le référentiel départemental existe.
4. Si non, vérifier si l'adresse est renseignée.
5. Si oui, vérifier si le numéro de voie est renseigné.
6. Si le numéro de voie manque, lever un contrôle faible priorité
   sur la rue à vérifier manuellement.
7. Vérifier la cohérence du code département avec le pays
   quand la règle existe.


CE QUI DISPARAÎT PAR RAPPORT À L'ANCIENNE VERSION
-------------------------------------------------

- plus de logique de "site localisable / non localisable" sur plusieurs anomalies
  détaillées ;
- plus de rapport "1 ligne = 1 anomalie" ;
- plus de diagnostics avancés type inversion X/Y, signe erroné, fuzzy matching ville ;
- plus d'exclusion d'un site uniquement parce qu'un pays n'a pas tous les référentiels.


CE QUI EST CONSERVÉ
-------------------

- filtre sites actifs ;
- bbox pays ;
- bbox départementales / régionales quand elles existent ;
- génération d'un Excel.


SORTIE ATTENDUE
---------------

Le rapport principal doit être lisible par un métier :

- 1 ligne = 1 site ;
- un seul libellé d'anomalies consolidé ;
- des codes simples ;
- une priorité claire ;
- un statut clair.


COLONNES TECHNIQUES AJOUTÉES PAR LE PIPELINE
--------------------------------------------

_SITE_KEY                  → Clé métier observée : NU_CNT + ID_SITE
_CONTROL_MODE              → GPS_OK / ADRESSE_OK / INCOMPLET
_CONTROL_STATUS            → OK / A_COMPLETER / A_CORRIGER / A_VERIFIER
_PRIORITY                  → HAUTE / MOYENNE / FAIBLE / OK
_ISSUE_COUNT               → Nombre d'anomalies consolidées sur le site
_ISSUE_CODES               → Codes anomalies séparés par virgule
_ISSUE_SUMMARY             → Libellé métier final

_HAS_GPS                   → GPS présent ou non
_GPS_IN_COUNTRY            → GPS dans la bbox pays
_GPS_MATCH_POSTAL_CODE     → GPS cohérent avec le CP quand vérifiable
_ADDRESS_PRESENT           → Adresse renseignée
_STREET_NUMBER_PRESENT     → Numéro de voie renseigné
_DEPARTMENT_CODE           → Code département extrait du CP
_DEPARTMENT_COUNTRY_OK     → Contrôle faible priorité


REMARQUE IMPORTANTE SUR L'IDENTIFIANT SITE
------------------------------------------

Dans l'échantillon analysé, ID_SITE seul n'est pas unique.
La clé stable observée est le couple :

    NU_CNT + ID_SITE

C'est cette logique qu'il faut garder pour toute consolidation future.

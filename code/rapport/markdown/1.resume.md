## Migration du Datamart Construction de SAS vers PySpark

## Résumé

Ce rapport présente la migration du datamart Construction d'Allianz France, historiquement développé en SAS, vers une architecture Python/PySpark sur Azure Databricks. Ce projet s'inscrit dans le programme SAS EXIST, initiative stratégique visant à moderniser progressivement l'ensemble des datamarts P&C de l'entreprise.

La démarche a débuté par une analyse exhaustive du code SAS existant, dix-neuf fichiers, environ quinze mille lignes, sans documentation initiale ayant conduit à la production d'une documentation fonctionnelle complète. Sur cette base, une architecture médaillon (Bronze/Silver/Gold) a été conçue et validée comme standard pour les prochaines migrations. L'ensemble des règles de gestion (72) et des sources de données (43) a été recensé dans deux fichiers Excel de traçabilité avant tout développement.

Trois pipelines Python ont été implémentés avec PySpark : gestion du portefeuille et des mouvements (PTF Mouvements), calcul des capitaux avec indexation FFB (Capitaux), et agrégation des primes (Émissions). La validation repose sur la comparaison des KPI entre les sorties SAS de référence et les sorties Python, sur dix visions mensuelles. La parité fonctionnelle est confirmée pour les pipelines PTF Mouvements et Capitaux. La validation des Émissions est en cours.

Ce prototype établit les fondations (code, documentation, méthodologie) pour la migration des quatre marchés restants (Auto, Habitation, Santé, Épargne). Les modules développés (lecteurs génériques, système de logging, configuration externalisée) sont directement réutilisables, réduisant significativement l'investissement nécessaire pour les migrations suivantes.
